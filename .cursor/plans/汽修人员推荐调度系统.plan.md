## 一、已敲定的需求与决策

- 移动端：**企业微信应用（H5）**，师傅扫码填资料、接派单、收通知都在企业微信内完成，用企业微信 JS-SDK 免登（无需记密码）。
- 通知：**仅企业微信应用消息推送**（第一版不加短信）。
- 企业微信：**已开通且有管理员权限**，可直接创建自建应用。
- 电脑端：**调度后台**，实时看板 + 智能推荐 + 一键派单，管理员账号密码登录（≤10 人，超管分配账号）。
- 派单逻辑：系统**自动推荐**最合适师傅（擅长部位匹配 + 空闲状态 + 接单量均衡），管理员确认后**一键派单**。
- 状态：空闲 / 工作中，进入工作中满 **5 小时自动转回空闲**。
- 评估重点：**擅长部位分布统计** + **实时调度大屏**。
- 部署：**阿里云 ECS 云端**运行（我带你买服务器、配域名、走备案）。预算按性价比选型。
- 位置/距离：本期仅**预留字段**，不做功能。
- 两个独立网址：`admin.你的域名`（后台）与 `m.你的域名`（移动端），API 走 `api.你的域名` 或 `/api`。

## 二、技术选型（我已定）

- 后端：**NestJS（Node.js + TypeScript）**
- 数据库：**PostgreSQL**，**Redis**（状态、定时）
- 实时：**Socket.IO（WebSocket）**
- 电脑后台：**React + Vite + Ant Design + ECharts**
- 移动端：**React + Vite + Vant**
- 部署：**ECS + Docker Compose** + 域名备案 + HTTPS。规格建议 2 核 4G。

## 四、数据模型（核心表）

- `admin_user`：管理员（账号、密码哈希、角色 超管/调度员）
- `worker`：师傅档案（姓名、**手机号(重点显示)**、企业微信 userid、头像、擅长部位[多选]、擅长品牌[多选]（含深向/零一/宁德时代）、状态 空闲/工作中、工作中开始时间、简介、位置字段[预留]）
- `skill_category`：部位分类（一级/二级，可后台维护）
- `dispatch_order`：派单（车辆品牌、故障部位、描述、推荐师傅、实际派给谁、状态、时间）
- `notification`：通知记录
- `stat_event`：行为/统计事件（预留点击分析）

## 五、功能模块

### 移动端（企业微信 H5）
- 企业微信免登；首次扫码填写/完善档案，实时保存
- 空闲/工作中一键切换；工作中显示时长
- 派单列表：接单/完成
- 通知中心：企业微信推送 + 应用内列表

### 电脑后台（调度系统）
- 账号密码登录
- 实时调度大屏（WebSocket + ECharts）
- 师傅档案管理（突出联系方式与擅长部位）
- 智能派单（品牌含深向/零一，部位含宁德时代电池/三电）
- 统计分析、分类维护、管理员管理

### 自动化
- 工作中满 5 小时自动转空闲

## 六、运行方式说明（已确认）

**上线后：本地电脑不需要 24 小时开机，也不需要在本地运行任何服务。**

| 阶段 | 本地电脑 | 阿里云 ECS |
|------|----------|------------|
| **开发调试期** | 可选：本地临时跑服务做联调（开发完可关掉） | 备案完成后部署测试环境 |
| **正式上线后** | **无需运行任何服务**，关机也不影响系统 | **24 小时持续运行**全部服务 |

**云端 24 小时运行的服务（全部在 ECS 上）：**
- Nginx（网关 + HTTPS）
- NestJS 后端 API + WebSocket
- **PostgreSQL 数据库**（师傅档案、派单、统计等全部数据，存在云端）
- **Redis**（在线状态、定时任务缓存，也在云端）
- 电脑后台前端静态文件
- 移动端 H5 静态文件
- 5 小时自动转空闲的定时任务

> **数据库说明**：PostgreSQL 和 Redis 与后端一起部署在同一台阿里云 ECS 上（Docker Compose），数据持久化在云服务器的磁盘里。你本地电脑不存任何业务数据。后续若数据量增大，可升级到阿里云 RDS（托管数据库），第一版用 ECS 内置方案即可，成本更低。

**本地电脑上线后仅用于（可选，非必须）：**
- 浏览器访问 `admin.域名` / `m.域名` 使用系统
- SSH 登录服务器做维护（或我帮你写部署脚本，你偶尔点一下更新）
- GitHub 上看代码、提需求
